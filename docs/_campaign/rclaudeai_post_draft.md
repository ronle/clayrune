## Body

**What it is**

Clayrune is a console that sits on top of Claude Code and runs all your agents from
one place. Every project is a tile on one grid, so you can see what each agent is
doing at a glance instead of digging through a pile of terminal windows.

What it does:

* Run agents across multiple projects in parallel, all visible on one dashboard
* Cross-session memory, so agents don't start cold or re-read the whole project to catch up
* A backlog per project for the ideas you don't want to lose
* Scheduled and recurring work, plus multi-agent runs
* Check in — and answer an agent's question — from your browser or your phone

The problems it solves, all of which I actually hit:

* Six terminal windows and no idea which one was doing what
* Agents with no memory of the last session, re-analyzing the project every time
* Re-explaining the same context and decisions over and over
* Notes, docs and ideas scattered across projects

It's free and MIT, and it runs on your machine with your Claude subscription —
nothing leaves your box. Honest caveat: because it runs on your machine, if the
machine sleeps the agent stops (Clayrune tells you it did). No hosted Claude, no
"always on."

**Why I built it**

I got laid off last August and went all-in building my own thing with AI. That grew
into a handful of projects, and I ended up running Claude Code in five or six
terminal windows at once. Every morning I'd come back and not remember which agent
window was doing what. Some had no memory of the previous session and re-read the
whole project to catch up; with others I had to walk back through the project steps
just to remind the agent where we were. Notes scattered everywhere. I was spending
more time coordinating, memorizing and managing agents than actually building.

So I built the project-management tool that made the most sense to me — and I built
the whole thing with Claude Code itself. Claude did most of the heavy lifting: I'd
describe the behaviour I wanted, it wrote the backend and the frontend, and I
reviewed and steered. The memory system and the multi-agent orchestration were both
designed over long back-and-forth sessions with it.
